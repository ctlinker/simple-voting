# Note sur les extensions PHP

## Extensions PHP

Les extensions PHP définissent ce que votre commande `php` peut faire, y compris les fonctions et classes globales disponibles. Elles permettent également d'ajouter des fonctions et classes personnalisées utilisables dans vos scripts PHP.

Les extensions PHP courantes incluent :

* **mysql** : Fonctions client MySQL (obsolètes dans les versions modernes de PHP ; utilisez `mysqli` ou `PDO_MySQL` à la place)
* **pdo** : Objets de données PHP (une couche d'abstraction pour l'accès aux bases de données)
* **pdo_mysql** : Pilote PDO pour les bases de données MySQL
* **iconv** : Conversion internationale de jeux de caractères

…et bien d'autres.

Composer vous permet d'installer des bibliothèques et de gérer les dépendances pour vos projets PHP. Gardez à l'esprit que certaines bibliothèques dépendent d'extensions PHP spécifiques pour fonctionner correctement.

La plupart des extensions PHP sont installées par défaut, mais certaines peuvent nécessiter une installation manuelle. Vous pouvez vérifier quelles extensions sont installées en exécutant :

```bash
php -m
```

Certaines extensions peuvent également devoir être activées dans votre fichier `php.ini`. L'emplacement de `php.ini` dépend de votre système :

* **Linux** :
  `/etc/php/7.4/cli/php.ini`
  `/etc/php/7.4/apache2/php.ini`
  `/etc/php/php.ini`

* **Windows** :
  `C:\xampp\php\php.ini`
  `C:\wamp\bin\php\php7.4.12\php.ini`
  `C:\wamp64\bin\php\php7.4.12\php.ini`

* **macOS** :
  `/usr/local/etc/php/7.4/php.ini`
  `/usr/local/etc/php/php.ini`

Si votre projet dépend d'une extension PHP spécifique, vous pouvez la déclarer dans votre fichier `composer.json` sous la section `require`. Par exemple :

```json
"require": {
    "ext-iconv": "*",
    "ext-gd": "*",
    "ext-mysqli": "*",
    "ext-openssl": "*",
    "ext-pdo": "*",
    "ext-pdo_mysql": "*"
}
```

Cela garantit que toute personne installant votre projet via Composer sera informée si une extension requise est manquante.
